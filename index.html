Balancing Brackets

For a given input string of open and closed brackets, determine if the set is closed or not closed.

Ex: "( )" is true

Before you even THINK about coding, analyze the problem:

  • Define what closure means
  • What's the input? (string of text)
  • What's the output? (true or false)
  • Define rules
  • Break out rules into test cases
  • What tools can you use for the job?
  • Now you can build a solution

Input is a string of brackets

Output is true or false

THEREFORE: the main operation must be a comparison

What are the rules?:

  • Closures happen in pairs --> string must be even in length
  • Open bracket must occur before a closed bracket --> order matters

Breakout rules into test cases:

Make sure to list cases that are true and false

  • "()" is true
  • "())" is false # odd
  • "()((" is false # not closed
  • "(()())" is true
  • ")(" is false # closed bracket before open

What tools can be used for the job?

Order Matters:

  • Counter
  • Array

Adding complexity can change your solution

  • What if there are multiple types of brackets?
  • What happens if the input is extremely large?
  • What happens if there is other characters in the input?

Typically the best solution performs the best for all complexities of the problem.

This is where things can get more complex...

So how do we determine the best answer?

Good Question!

Big-O notation can help!

Big-O notation describes:

  • The complexity of the operations (computationally) proportional to the size of the input
  • The efficiency of an computation
  • What the worst-case scenario is for an operation

Big-O descriptors (where N is the size of the input):

  • O(1) == Constant time
  • O(N) == Linear time
  • O(NlogN) == Logorithmic time
  • ...
  • O(2^N) == Exponential time
  • ...etc...
